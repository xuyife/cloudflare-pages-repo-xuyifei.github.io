1.GithubFast国内镜像失效

最近很多网友应该都发现了我之前推荐的githubfast.com国内github镜像失效了，打开之后会出现一行文字：

翻译过来大概就是：为了防止滥用，请使用正版github。目前没有解决办法。

但是我们可以使用加速器加速github.com的访问。

2.Github加速

接下来我来介绍几种加速方法

注：本教程只适配Windows系统

2.1.SwitchHosts加速（推荐）

switchHosts通过修改Hosts文件把Github相关链接重定向到更快的Github的IP地址

2.1.1修改Hosts文件权限

Hosts文件所在文件夹路径如下：

使用文件资源管理器打开此目录

在这里你将看到hosts文件，右键该文件，点击属性，点击“安全”，编辑权限

点击“编辑”，编辑“Users”的权限

增加修改和写入的权限，点击确定。

点击确定后可能会跳出安全提示，点击“是”即可

回到“常规”一栏，取消勾选“只读”

2.1.2下载软件-SwitchHosts

这里是SwitchHosts的GitHub发布页链接，您可以选择在发布页下载最新版本：

https://github.com/oldj/SwitchHosts/releases

但为了避免“在GitHub解决如何访问GitHub”的尴尬局面，我向大家提供SwitchHosts Windows 4.2.0 Beta版本的下载链接：

提取码：6666

2.1.3安装并使用SwitchHosts

点击SwitchHosts的安装包，执行安装步骤（可以安装在系统盘以外的盘符），就可以安装SwitchHosts软件了。

安装完成后，打开SwitchHosts，点击左上角的”+“号：

点击”远程“，如图填入相关信息：

Hosts类型选择远程，标题可随便起，URL如下：

根据作者的更新频率，建议您将自动刷新时间调整为1小时。

随后点击确定，再启动规则：

若出现”操作成功“字样，则GitHub加速成功！
